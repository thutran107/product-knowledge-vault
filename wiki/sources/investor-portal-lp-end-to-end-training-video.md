## Summary
A **how-to-demo** walkthrough of the investor-facing platform end to end (using a "Nucleus Capital" sandbox): branded home page, incoming-opportunity landing pages, permission-gated data rooms, an embedded fund-subscription widget, and the post-close hub (reports, documents, inbox, profile). Explicitly framed as a sales tool — guide a prospect through a curated narrative, not an exhaustive feature tour.

## Key Takeaways
- **Home page does IR work pre-raise:** surfaces *incoming* (not-yet-open) funds to warm the pipeline without direct outreach.
- **Permission-gated data room** auto-notifies investors when their access level changes — no manual follow-up; embedded document highlights create a curated first impression.
- **Embedded subscription widget** on the opportunity landing page collapses the interested→subscribed step (no context switch). **Co-branding** for placement agents / wealth advisors extends white-labeled experiences to intermediary channels.
- **Post-close = one destination** (reports, documents, inbox, profile) replacing scattered emails and portal-per-fund.
- **Reporting — set expectations honestly:** dashboards are **tables-only** today (customizable columns, multiple datasets, quarterly + interim in one view); **all performance metrics must be imported, not computed**; **charts targeted for Q3**, metric computation later. "Lead with flexibility and set clear expectations."
- **Documents tab:** all post-close docs (K-1s, distribution notices, quarterly reports) filterable by entity/type/fund/date; nearly all file types — including **video and audio** — previewable in-portal.
- **Inbox** is a read-only log today; roadmap is a two-way communication hub (inquiries, comments, messages in one thread).
- **Profile/KYC view-only today**; upcoming workflow lets investors submit edits + manage their own team's contact-matrix access, subject to GP/fund-admin review.
