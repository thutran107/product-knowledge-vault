## Summary
A step-by-step **demo-prep** walkthrough: rebrand the default sandbox to look like a specific prospect before a client call (logo, nav, global colors, images, system-page theming). Reinforces the landing-page builder mechanics and one critical workflow habit — **publish before navigating away**.

## Key Takeaways
- **Set global colors first** (three Primary slots in Site Settings) — linked widgets across every page pick up the palette automatically, eliminating per-page recoloring. "Highest-leverage step."
- **Nav bar / Design tab:** replace logo, set max-height, **hide irrelevant menu items** (control the demo narrative), update active-menu colors.
- **Images optional:** when client imagery is unavailable, remove hero/cover images and use a **solid brand-color section** — clean and on-brand without assets.
- **Fund pages rebrand themselves** — opening a fund page shows the global color scheme already applied (system coherence; stage this in a live demo). Pre-built advisory/FAQ/reporting pages exist as ready assets.
- **System pages need separate theming:** a dedicated color-theme modal (System Pages tab) — enter the primary brand color, auto-generates complements; **review readability/contrast** (manual override toggle exists).
- **Browser tab:** optional page-name update — visible on screen-share/screenshot.
- **Publish before switching pages:** unsaved changes are **silently discarded** when you leave the builder — the single most important habit.
- **Final LP-view review:** walk the investor path to catch inconsistencies before the call.
